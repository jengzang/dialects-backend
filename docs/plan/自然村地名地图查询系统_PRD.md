# 1. 项目概述

## 1.1 背景

本系统面向自然村地名数据的空间展示与查询需求，建设一个基于 GIS 地图的自然村地名浏览平台。

系统通过地图展示自然村空间分布，支持用户查看自然村位置、搜索自然村名称，并在用户点击具体地名后调用民政部官方地名服务获取详细信息。

## 1.2 产品目标

实现：

1. 自然村空间可视化展示
2. 自然村名称搜索
3. 地名点位交互查询
4. 对接民政部官方地名详情服务
5. 为后续地名文化分析、空间统计分析提供基础平台

# 8. 后端 API

## 8.1 获取全部自然村点

接口：

```
GET /api/toponyms/map
```

用途：

地图初始化。


---

## 请求参数

无。


原因：

第一版本直接返回全部自然村。


---

## 返回字段

仅返回地图必要字段：

|字段|说明|
|-|-|
|id|地名ID|
|name|名称|
|longitude|经度|
|latitude|纬度|


---

## 返回示例

```json
{
"type":"FeatureCollection",
"features":[
{
"type":"Feature",
"geometry":{
"type":"Point",
"coordinates":[113.1,23.2]
},
"properties":{
"id":"123",
"name":"黄村"
}
}
]
}
```

---

# 9. 数据传输优化

## 9.1 字段裁剪

地图阶段不返回：

- 行政区
- 历史名称
- 分类信息
- 分析字段


只返回：

```
id
name
longitude
latitude
```

---

## 9.2 HTTP压缩

推荐开启：

- gzip
- brotli


浏览器自动解压。


---

## 9.3 未来优化

如果数据规模扩大：

可考虑：

- protobuf
- msgpack
- 矢量瓦片


第一版无需实现。

# 12. 民政部接口说明


接口地址：

```
https://dmfw.mca.gov.cn/9095/stname/detailsPub
```


请求方式：

```
POST
```


---

## 请求参数

|参数|必须|类型|说明|
|-|-|-|-|
|id|是|String|地名ID|


示例：

```json
{
"id":"123456"
}
```


---

## 返回字段


|字段|类型|说明|
|-|-|-|
|area_name|String|所在区县名称|
|city_name|String|所在地市名称|
|old_name|String|历史地名|

# 20. Agent开发说明

## 后端 Agent

请探索已有数据库中的 toponyms 表。

目标：

实现一个地图初始化接口。

要求：

- 返回全部自然村
- 只返回 id/name/经纬度
- 评估数据量
- 开启压缩
- 不实现详情接口


---

## 前端 Agent

实现自然村地图页面。

要求：

- 使用 MapLibre
- 加载全部自然村点
- 点击 marker 获取 id
- 直接调用民政部接口
- 展示详情
- 搜索使用前端过滤
