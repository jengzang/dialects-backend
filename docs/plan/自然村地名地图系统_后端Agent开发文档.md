# 自然村地名地图系统 - 后端 Agent 开发文档

## 1. 任务目标

你负责实现自然村地名地图查询系统的后端接口。

核心目标：

> 基于已有 toponyms 数据表，为前端地图提供自然村基础空间数据。

注意：
- 不负责数据分析
- 不负责创建数据库表
- 不负责民政部地名详情接口
- 不负责前端搜索逻辑


---

# 2. 已有条件

数据库：

```
toponyms
```

该表已经由数据分析团队完成。

开发前必须先探索：

1. 表结构
2. 字段名称
3. 数据量
4. 经纬度字段
5. 地名 ID 字段


不要假设字段名称。


---

# 3. 核心接口

## GET /api/toponyms/map


用途：

地图初始化时获取全部自然村点。


---

# 4. 接口要求


## 请求

无参数。


原因：

第一版采用一次性加载全部自然村数据。


---

## 返回字段

仅返回地图必须字段：


|字段|说明|
|-|-|
|id|地名唯一ID|
|name|自然村名称|
|longitude|经度|
|latitude|纬度|


不要返回：

- 详细行政区
- 历史名称
- 分析字段
- 大字段


---

# 5. 返回格式


推荐：

GeoJSON


示例：

```json
{
"type":"FeatureCollection",
"features":[
{
"type":"Feature",
"geometry":{
"type":"Point",
"coordinates":[113.1,23.2]
},
"properties":{
"id":"123",
"name":"黄村"
}
}
]
}
```


原因：

MapLibre 可以直接加载。


---

# 6. 性能要求


## 数据量评估

实现前先统计：

- 总记录数量
- 返回 JSON 大小


---

## 压缩

必须开启：

- gzip

或：

- brotli


确保浏览器自动解压。


---

# 7. 数据查询要求


SQL 查询只获取必要字段。


不要：

```
SELECT *
FROM toponyms
```


推荐：

```
SELECT
id,
name,
longitude,
latitude
FROM toponyms
```


---

# 8. 索引检查


检查：

- id 是否有索引
- 经纬度字段类型


虽然第一版全量读取，但需要了解数据情况。


---

# 9. 不需要实现的功能


以下不要实现：

## 地名详情接口

错误：

```
GET /api/toponyms/{id}
```


原因：

详情由前端直接请求民政部。


---

## 搜索接口

错误：

```
GET /api/toponyms/search
```


原因：

第一版前端本地搜索。


---

## bbox动态加载

错误：

```
GET /api/toponyms/map?bbox=
```


原因：

第一版直接加载全部点。


---
